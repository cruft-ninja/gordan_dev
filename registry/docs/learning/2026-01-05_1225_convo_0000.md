To create a custom slash command for the Gemini CLI, you need to define it in a `.toml` file.

I have set up a project-scoped example for you. Here is the step-by-step guide:

1.  **Create the Commands Directory**:
    Custom commands are stored in `.gemini/commands/`.
    *   **Global**: `~/.gemini/commands/` (available everywhere).
    *   **Project**: `.gemini/commands/` (available only in this folder).
    I created the local directory for you: `.gemini/commands/`

2.  **Create the Command File**:
    The filename becomes the command name. For example, `hello.toml` becomes `/hello`.
    I created a sample file `.gemini/commands/hello.toml` with the following content:

    ```toml
    description = "A friendly greeting command"
    prompt = "Say hello to {{args}} in a pirate style!"
    ```

3.  **How to Use It**:
    You can now use this command in the Gemini CLI by typing:
    `/hello World`
    (The `{{args}}` placeholder will be replaced by "World", resulting in the prompt: "Say hello to World in a pirate style!")

**Key Features:**
*   **Arguments**: Use `{{args}}` to insert text typed after the command.
*   **Shell Commands**: Use `!{command}` to run a shell command and insert its output (e.g., `Summarize this git status: !{git status}`).
*   **Namespaces**: Subdirectories create namespaced commands (e.g., `commands/git/commit.toml` -> `/git:commit`).
